# Multi-Agent Code Review System

이 프로젝트는 세 개의 전문 에이전트가 병렬로 코드를 검토하는 자동화된 리뷰 시스템입니다.

## 코드 리뷰 워크플로우

PR이 열리거나 커밋이 푸시되면 오케스트레이터가 아래 순서로 리뷰를 수행합니다.

### 1단계 — diff 수신 및 파싱

- GitHub PR에서 변경된 파일 목록과 diff를 수집합니다.
- 분석 대상 파일을 확장자 기준으로 필터링합니다.

### 2단계 — 에이전트 병렬 실행

세 에이전트를 `isolation: worktree`로 독립 실행합니다.

| 에이전트 | 파일 | 담당 영역 |
|----------|------|-----------|
| security-reviewer | `.claude/agents/security-reviewer.md` | SQL Injection, XSS, 시크릿 노출, 명령어 인젝션 |
| performance-reviewer | `.claude/agents/performance-reviewer.md` | N+1 쿼리, 무한루프, 메모리 누수, 알고리즘 복잡도 |
| style-reviewer | `.claude/agents/style-reviewer.md` | 네이밍 컨벤션, 코드 중복, 포맷, 주석 품질 |

### 3단계 — 결과 취합 및 PR 코멘트

세 에이전트의 결과를 아래 기준에 따라 취합한 뒤 `gh pr comment`로 GitHub PR에 게시합니다.

#### 취합 기준 1 — 중복 이슈 제거

동일 파일·라인을 지적하는 이슈가 여러 에이전트에서 중복 발견된 경우, **가장 높은 심각도의 이슈만 유지**하고 나머지는 제거한다.

예시: `storage.js:27`에 대해 security-reviewer가 [중간], performance-reviewer가 [높음]으로 지적한 경우 → [높음] 이슈만 유지.

#### 취합 기준 2 — 우선순위 매트릭스

전체 이슈를 수정 난이도(쉬움/중간/어려움)와 영향도(높음/중간/낮음) 기준으로 분류하여 아래 매트릭스를 작성한다.

```
             영향도 높음   영향도 중간   영향도 낮음
난이도 쉬움  [ P1 즉시처리 ] [ P2 다음스프린트 ] [ P3 여유시처리 ]
난이도 중간  [ P1 즉시처리 ] [ P2 다음스프린트 ] [ P3 여유시처리 ]
난이도 어려움[ P2 다음스프린트] [ P3 여유시처리 ] [ P4 백로그    ]
```

- **P1**: 즉시 처리 — 현 PR 머지 전 수정 권고
- **P2**: 다음 스프린트 처리
- **P3**: 여유 시 처리
- **P4**: 백로그 등록

#### 취합 기준 3 — 높음 심각도 요약 섹션

`final.md` 상단에 높음 심각도 이슈만 추출한 별도 요약 섹션을 추가한다.

```markdown
## ⚡ 즉시 조치 필요 (높음 심각도)

| # | 파일:라인 | 이슈 | 분류 | 우선순위 |
|---|-----------|------|------|----------|
| 1 | 파일명:라인 | 이슈 제목 | 보안/성능/스타일 | P1/P2 |
```

높음 심각도 이슈가 없으면 해당 섹션을 생략한다.

## 출력 형식

```
## 코드 리뷰 결과

🔴 높음 N건 / 🟡 중간 N건 / 🟢 낮음 N건

### 🔴 높음
[높음] 파일명:라인번호 - 취약점 제목
  → 설명: ...
  → 권고: ...

### 🟡 중간
[중간] 파일명:라인번호 - 문제 제목
  → 설명: ...
  → 권고: ...

### 🟢 낮음
[낮음] 파일명:라인번호 - 개선 제목
  → 설명: ...
  → 권고: ...
```

## 결과 파일 저장 규칙

각 에이전트는 분석 완료 후 결과를 아래 경로에 Markdown 파일로 저장한다.

| 에이전트 | 저장 경로 |
|----------|-----------|
| security-reviewer | `/tmp/review/security.md` |
| performance-reviewer | `/tmp/review/performance.md` |
| style-reviewer | `/tmp/review/style.md` |

오케스트레이터는 세 파일이 모두 생성된 것을 확인한 뒤 내용을 읽어 심각도 순(높음 → 중간 → 낮음)으로 취합하고, 최종 리뷰를 `/tmp/review/final.md`에 작성한다.

```
/tmp/review/
├── security.md       ← security-reviewer 결과
├── performance.md    ← performance-reviewer 결과
├── style.md          ← style-reviewer 결과
└── final.md          ← 오케스트레이터 최종 취합 결과
```

- 에이전트 실행 전 `/tmp/review/` 디렉토리가 없으면 생성한다.
- 각 에이전트는 기존 파일이 있으면 덮어쓴다.
- `final.md` 작성 후 `gh pr comment` 명령으로 GitHub PR에 게시한다.

## 에이전트 추가 지침

- 각 에이전트는 할당된 영역만 검사합니다. 범위를 벗어난 지적은 하지 않습니다.
- 오탐(false positive)보다 미탐(false negative)을 더 경계합니다.
- 수정 권고는 구체적인 코드 예시와 함께 제시합니다.
